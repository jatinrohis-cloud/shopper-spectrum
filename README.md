# Shopper Spectrum
### Customer Segmentation and Product Recommendation in E-Commerce

This project analyzes online retail transaction data to segment customers
(RFM + clustering) and recommend similar products (item-based collaborative
filtering), with a Streamlit app to try both interactively.

## Project structure

```
shopper-spectrum/
├── data/
│   ├── online_retail.csv        # raw dataset (place your CSV here)
│   ├── cleaned_data.csv         # generated by data_cleaning.py
│   └── rfm_segments.csv         # generated by rfm_segmentation.py
├── models/                      # generated .pkl artifacts (scaler, kmeans, similarity matrix)
├── outputs/figures/             # generated EDA & clustering charts
├── src/
│   ├── data_cleaning.py         # Step 1: clean the raw data
│   ├── eda.py                   # Step 2: exploratory data analysis
│   ├── rfm_segmentation.py      # Step 3: RFM + KMeans customer segmentation
│   └── recommendation.py        # Step 4: item-based product recommender
├── app.py                       # Step 5: Streamlit application
├── requirements.txt
└── README.md
```

## Setup (VS Code)

1. Open this folder in VS Code (`File > Open Folder...`).
2. Create and activate a virtual environment (optional but recommended):
   ```bash
   python -m venv .venv
   # Windows
   .venv\Scripts\activate
   # macOS/Linux
   source .venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Make sure `data/online_retail.csv` contains the raw dataset (it's already
   included if you copied this project as-is).

## Run the pipeline

Run each step from VS Code's integrated terminal, in order, from the project
root folder:

```bash
python src/data_cleaning.py       # -> data/cleaned_data.csv
python src/eda.py                 # -> outputs/figures/*.png
python src/rfm_segmentation.py    # -> data/rfm_segments.csv, models/*.pkl
python src/recommendation.py      # -> models/product_similarity.pkl, product_lookup.pkl
```

Each script prints progress and summary stats to the terminal so you can
follow along without a notebook.

## Run the Streamlit app

```bash
streamlit run app.py
```

This opens a local web app with two tabs:
- **Product Recommendation** — pick a product, get 5 similar product suggestions.
- **Customer Segmentation** — enter Recency/Frequency/Monetary, get the
  predicted segment (High-Value / Regular / Occasional / At-Risk).

## Notes on methodology

- **Data cleaning**: drops rows with missing `CustomerID`/`Description`,
  removes duplicates, removes cancelled orders (`InvoiceNo` starting with
  `C`), removes non-positive `Quantity`/`UnitPrice`, and strips extra
  whitespace from all text columns.
- **RFM**: Recency = days since last purchase (relative to the day after the
  last invoice date in the data); Frequency = number of distinct invoices;
  Monetary = total spend.
- **Segmentation**: Frequency and Monetary are log-transformed to reduce
  skew, then all three RFM features are standardized before KMeans
  (k=4, chosen via elbow/silhouette — see `outputs/figures/elbow_plot.png`).
  Clusters are auto-labeled into business names based on their RFM profile
  (best combined rank = High-Value, worst = At-Risk).
- **Recommendation**: builds a Customer × Product (StockCode) matrix of
  quantities purchased, then computes item-item cosine similarity. Given a
  product, the top-5 most similar products (by purchase co-occurrence
  pattern) are returned.
